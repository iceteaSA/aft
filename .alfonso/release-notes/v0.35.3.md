# v0.35.3 — LSP auto-install reliability

A small patch fixing a silent LSP auto-install failure and a README rendering bug.

## LSP servers install reliably again

AFT installs language servers with `npm install --no-save` into its own cache directory. When that directory has no `package.json`, npm walks up the directory tree, and if any parent (for example a `package.json` in your home directory) has one, npm installs into that package's `node_modules` instead. The cache directory ends up empty while npm still exits successfully, so the install silently fails and you see a recurring "LSP binary missing" warning even with `lsp.auto_install` enabled. This surfaced most on newer npm versions.

AFT now writes a small anchoring `package.json` into the cache directory before installing, so the server lands where AFT expects it. This affects both OpenCode and Pi.

## README

Fixed the YAML row in the Supported Languages table (it had the wrong number of columns) and a missing blank line that made a horizontal rule render as a heading.
